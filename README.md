# Package Reviews

The [Fedora Package Review process][process] is required to add packages to Fedora.
However, this process typically involves performing an iterative review via bugzilla comments.
It would be nice to be able to do review with modern code review tooling,
such as what is provided here in GitHub.

This repo exists to experiment with what a package review process would look like
if it were done via pull request with modern code review tools.
I plan to work with new contributors to get their new spec files in good shape here,
before submitting them for an official Fedora review in bugzilla,
at which point the resulting spec files should be in a state that they can be immediately approved.

See also:

* [Fedora Package Review Guidelines][guidelines]
* [Fedora Package Review Policy][policy]
